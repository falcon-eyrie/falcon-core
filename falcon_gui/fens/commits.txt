5b6aa4e | refactoring of shared state system
ProcessorGraph now manages a map of state aliases that represent groups of shared processor states.
Shared states agree on an external read/write permission that is compatible with all.
Shared states cache their value locally on each set/get operation, so that processors can easily check
if a state value was changed by another processor (or externally)

use shared_ptr everywhere for both IState and the shared atomic value
Readable and Writable State classes are now clonable, and protect internals with locks for thread-safe access

exported yaml of states takes external permission into account
added 'y' keyboard command to check exported yaml description of graph

state update/retrieval methods removed from IProcessor, so that IProcessor does not have to be friend of IState
IProcessor create state methods have additional description argument

methods added to ProcessorGraph to look up processors and states by name
ProcessorGraph state update/retrieve methods now also deal with state aliases
